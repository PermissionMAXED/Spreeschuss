# Family VFX — Holosphere Visuals & Dreamscape Cartridges (`holo_projection`)

Global sequence 274–300 · 27 features (20 core / 7 stretch) · core wave W13, stretch W15 · catalog family string: `holo_projection` · split: 10 lens items + 11 cartridge scene items + 6 support systems (3 core blocks/tools: VFX-22/23/24; 3 stretch: VFX-25/26/27).

Cosmetic visual layer for the U23 Holosphere Dreamscape Projector (binding spec: `docs/expansions/CP0C_HOLOSPHERE.md`; U23 holds global sequence 273, directly before this family): lenses skin the dome surface, cartridges project interior dreamscapes, support systems only drive state consumed by the two passes. Every entry is an independently acquired, player-facing feature — an item or block with its own survival acquisition loop (recipe, mob drop, structure loot, trade, milestone unlock, or fishing), its own per-feature acquisition advancement `cuprum:dreams/vfx_NN` granted exactly once on first obtain (craft, pickup, trade, or unlock — asserted in every row's acceptance), its own handbook page (`handbook.cuprum.vfx_*`, EN/DE parity), and a distinct runtime behavior. None is a mere preset of another. All projected content is client-side illusion only: render state, never `Entity` objects — no hitboxes, no AI, no per-entity sync, zero server tick cost, 0 items and 0 saturation from any interaction.

**Balance constants:** lens +2 Cg/t, cartridge +4 Cg/t on U23 upkeep (base 4/9/16/36 at R8/12/16/24); blank cartridge = 4 copper ingots + 2 glass + 1 redstone + 1 amethyst shard; lens blank = cast at an OXI-15 lens mold from 2 copper ingots + 1 glass; presets quantized (4 density, 3 speed, 16 palettes) stored as data components.
**Performance budget:** frozen shader inventory — exactly 2 RenderTypes family-wide: `cuprum:holo_surface` (RenderPipeline resource id `cuprum:pipeline/holo_surface`, consuming the reserved dome slot; one übershader whose fragment dispatch reads a packed variant id carried in vertex attributes, covering all 10 lens variants VFX-01..10) and `cuprum:holo_interior` (RenderPipeline resource id `cuprum:pipeline/holo_interior`, consuming the reserved aurora slot; one übershader that mode-switches between far-scene, streak and 2.5D/low-poly holo content, covering all 11 cartridge scenes VFX-11..21), plus the shared original include library `assets/cuprum/shaders/include/cuprum_noise.glsl` (`#moj_import`); the 6 support systems (VFX-22..27) add zero pipelines; WEA-13 at T1 reuses the `holo_interior` far-scene mode on the same reserved aurora slot and never registers a fifth RenderType. Per visible R12 projector (one lens + one active cartridge only — scenes never run concurrently): one no-cull additive surface geometry callback ≤4,096 verts + one interior geometry callback ≤8,192 verts, ≤2 geometry callbacks total per frame; T1 interiors render only for the 4 nearest visible projectors; HOLO particle sub-pool ≤32 spawn/tick and ≤128 live, carved from the existing 64/256 family totals (never additive to them); W14 reference scene (one R12 projector + one lens + one cartridge + the existing maximum shield ripple state) keeps the Holosphere contribution ≤1.5 ms/frame, gated by `w14_holo_frame_budget`, which also measures actual batch/draw counts; flash rate ≤3 Hz everywhere via the code-enforced flash-cap governor; all Holosphere blending is additive only.
**Security/ownership contract:** every custom-GUI write uses a typed payload from the CP0C §2.2 ledger (per-payload byte/rate caps, range/menu/ownership/state checks, bounded structured fields) carrying only bounded settings/indices and an exact relink `BlockPos` — never lens/cartridge/item content ids; the server derives all active content from authoritative container inventory, rejecting empty, wrong-type, or locked/unrecipe-gated slots; VFX-24 cycles palettes through the guarded vanilla `useOn` path (no payload, 5-tick per-player cooldown); VFX-26 bitmap ≤128 bytes + title ≤32 Unicode scalar values and ≤64 UTF-8 bytes (NFC-normalized; every `Bidi_Control=Yes` code point, control character, and noncharacter rejected; opaque display data only), owner-only; no C2S ever originates from render code; permission node `cuprum.holo.configure` (owner/OP fallback).
**Persistence contract:** sockets/programs/presets/`fxSeed`/controller pos/phase fields (`phaseTicksAccumulated`, `phaseStartedAtGameTime`, `phaseRunning`) live in the versioned `cuprum_state` BE envelope and item data components; identical after serialization round-trip and after the U23-level process-restart probe; animation freezes while PAUSED and resumes with no phase jump.
