# Family TUB — Pneumatic Tube Logistics (`tube_logistics`)

Global sequence 95–108 · 14 features (12 core / 2 stretch) · core wave W8, stretch W15 · catalog family string: `tube_logistics`

Routing, throughput and safety extensions of U11's visible pneumatic tubes. Items travel as virtual stacks on graph edges server-side; visible in-glass items are client interpolation from batched payloads.

**Balance constants:** base tube speed 4 blocks/s; one accelerator doubles a run to 8 blocks/s, capped at 16 blocks/s (×4); base network limit 256 items in flight, compressor raises it to 512; cross-dimension transfers (TUB-14) cost 1,000 Cg per capsule/stack.
**Performance budget:** ≤0.05 ms/tick per 100 tube segments; transit payloads batch at 4/s per network; the client renders at most 6 visible items per segment (overflow shown as a density tint).
**Item-safety contract (binding):** every routing operation is conservation-checked — items are never voided, duplicated or silently dropped; inventory mutations are atomic (remove-then-insert in one server tick or rolled back). Virtual stacks in unloaded chunks freeze in place and resume on load; on server restart, in-flight stacks are restored from the network SavedData.
**Cross-dimension transaction contract:** TUB-14 uses two-phase commit — the source removes an item only after the destination acknowledges reservation; a crash or unload mid-transfer leaves the item at the source on restart; sends are rate-limited to 10 transfers/s per nexus (excess queues, never drops).
**Cross-family interfaces:** tube ports are the standard item IO consumed by GOL-07 couriers, RAIL-06 sleds and machine GUIs; TUB-06 ejector launches items to entities and gaps; TUB-08 capsules appear in GEN-07 shipwreck loot.
